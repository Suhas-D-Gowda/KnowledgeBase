1. git init:
   used to initialize a folder with the GitHub (you get the gist of it)

2. git status:
   used to "check the status". Informs about any changes that have been made recently after the recent push.

3. git add: (git add filename/git add .)
   used to "add" the changes to the next push. This solidifies the changes that have been made and basically "includes what will be committed"

4. git commit -m "message":
   used to finalize the changes i.e. it like "sealing the box after "git add"ing to the box.

5. git push:
   used to "push" the code to GitHub (online).

6. git log:
   Shows the history of commits in the repository.

7. git config:
   used to configure Git's settings

8. git remote add origin:
   Connect local (NEW) repo to GitHub
